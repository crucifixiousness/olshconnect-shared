# OLSHCO System - Data Flow Diagram (DFD)

## System Overview
The OLSHCO (Our Lady of the Sacred Heart College) system is a comprehensive educational management platform that handles student enrollment, course management, grading, payments, and administrative functions.

## Context Level DFD (Level 0)

### External Entities
- **Students** - Individuals enrolled or seeking enrollment
- **Staff/Instructors** - Faculty and administrative personnel  
- **Administrators** - System administrators with full access
- **External Payment Systems** - Third-party payment processors
- **Document Storage** - File storage systems

### Central System
- **OLSHCO System** - Main educational management platform

### Data Flows
- Students ↔ OLSHCO System (enrollment, grades, payments, documents)
- Staff/Instructors ↔ OLSHCO System (course management, grading, schedules)
- Administrators ↔ OLSHCO System (system management, reports)
- External Payment Systems ↔ OLSHCO System (payment confirmations, transaction data)
- Document Storage ↔ OLSHCO System (document uploads, retrievals)

## Level 2 DFD - Detailed Process Breakdown

### 2.1 User Authentication & Authorization

#### Sub-processes
1. **P1.1: Input Credentials** - User enters username/password
2. **P1.2: Validate Credentials** - System checks against user database
3. **P1.3: Generate JWT Token** - Creates authentication token
4. **P1.4: Role Assignment** - Assigns user permissions
5. **P1.5: Log Failed Attempts** - Records unsuccessful login attempts

#### Data Flows
- Users → P1.1: Credentials
- P1.1 → P1.2: Credentials
- P1.2 → DS1: Query User Database
- P1.2 → P1.3: Valid Credentials
- P1.3 → P1.4: JWT Token
- P1.4 → DS2: Role Permissions
- P1.4 → Users: Access Granted
- P1.2 → P1.5: Failed Credentials
- P1.5 → DS3: Security Logs

#### Data Flows

## Data Dictionary

### External Entities
- **Students**: Individuals enrolled or seeking enrollment
- **Staff/Instructors**: Faculty and administrative personnel
- **Administrators**: System administrators with full access
- **Registrar**: Staff handling enrollment and academic records
- **Finance Staff**: Personnel managing financial transactions
- **Program Heads**: Academic leaders managing specific programs
- **External Payment Systems**: Third-party payment processors
- **Document Storage**: File storage systems

### Data Stores
- **Student Database**: Student profiles, personal information
- **Enrollment Database**: Enrollment applications and status
- **Course Database**: Course information and assignments
- **Financial Database**: Payment records and transactions
- **Academic Records**: Grades, transcripts, academic history
- **Document Database**: Student documents and forms
- **User Management**: User accounts and authentication
- **Security Logs**: System access and security events

### Key Data Flows
1. **Authentication Flow**: Credentials → Validation → Token Generation → Role Assignment
2. **Enrollment Flow**: Application → Verification → Status Update → Block Assignment
3. **Payment Flow**: Payment Submission → Processing → Confirmation → Balance Update
4. **Academic Flow**: Grade Input → Record Update → Transcript Generation
5. **Document Flow**: Request → Processing → Status Update → Delivery

## System Characteristics
- **Multi-role Access Control**: Different permission levels for different user types
- **Real-time Updates**: Live data synchronization across modules
- **Secure Authentication**: JWT-based token system with role-based access
- **Comprehensive Logging**: Audit trails for security and compliance
- **Modular Architecture**: Separate modules for different functional areas
- **API-based Communication**: RESTful API endpoints for data exchange

## Conversion Instructions

### For draw.io (diagrams.net)
1. Create a new diagram
2. Use the "Entity" shape for external entities (rectangles)
3. Use the "Process" shape for processes (circles/rounded rectangles)
4. Use the "Data Store" shape for data stores (open rectangles)
5. Use arrows for data flows
6. Add labels on arrows for data descriptions

### For Lucidchart
1. Create a new diagram
2. Use "Rectangle" for external entities
3. Use "Circle" or "Rounded Rectangle" for processes
4. Use "Open Rectangle" for data stores
5. Use "Connector" lines for data flows
6. Add text labels on connectors

### For Microsoft Visio
1. Create a new diagram
2. Use "Rectangle" shapes for external entities
3. Use "Circle" or "Rounded Rectangle" for processes
4. Use "Database" shapes for data stores
5. Use "Connector" lines for data flows
6. Add text callouts for data descriptions

### For Any Other Tool
- **External Entities**: Represent as rectangles
- **Processes**: Represent as circles or rounded rectangles
- **Data Stores**: Represent as open rectangles or cylinders
- **Data Flows**: Represent as arrows with labels
- **Use consistent colors** for different types of elements
